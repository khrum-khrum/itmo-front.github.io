Реляционные базы данных (SQL)

Реляционные базы данных (РБД) – это наиболее распространенный тип БД, основанный на структуре таблиц, связанных между собой. Язык SQL (Structured Query Language) используется для управления данными в РБД: добавления, удаления, изменения и извлечения информации. Популярные СУБД (системы управления базами данных) – MySQL, PostgreSQL, Oracle. Знание SQL – ключевой навык для работы с РБД.

NoSQL базы данных

NoSQL базы данных – это альтернатива РБД, предназначенные для работы с большими объемами данных и сложными структурами. Они предлагают гибкость в схеме данных и высокую масштабируемость. Существуют различные типы NoSQL: документные (MongoDB), графовые (Neo4j), колоночные (Cassandra). Выбор типа NoSQL зависит от специфики задачи и требований к хранению данных.

Нормализация и денормализация

Нормализация – это процесс организации данных в РБД для устранения избыточности и обеспечения целостности. Денормализация – обратный процесс, направленный на повышение производительности за счет добавления избыточности. Выбор между нормализацией и денормализацией зависит от приоритетов: целостность данных или скорость запросов. Понимание этих концепций важно для оптимизации БД.

Транзакции и ACID-свойства

Транзакции – это логическая единица работы с данными, которая либо выполняется полностью, либо не выполняется вовсе. ACID-свойства (атомарность, согласованность, изолированность, долговечность) гарантируют надежность транзакций. Понимание транзакций и ACID-свойств важно для обеспечения целостности данных в БД, особенно в многопользовательских средах, где несколько пользователей могут одновременно обращаться к данным.
